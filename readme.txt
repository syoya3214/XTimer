# XTimer
このプログラムはClaude AIで大体作りました
DQ10で使用することを想定しています

本プログラムは解析行為や自動操作などは行っておらず、透過ウィンドウを設置してタイマー等を動かし、一定の時間を記録していくだけのものです
したがって規約上問題あるものではないと考えていますが、使用に関しては自己責任でお願いします

設定は大体Config.iniか、文字上で右クリックすると出るコンテキストメニューで変更することができます

タイマーを戦闘中に表示されている時間に合わせて記録することで、次の規則的に現れるタイプのギミックを把握できるプログラムです
おまけでxlsxで表を表示したり、AIを手動でカウントできたりします
プリセットではルベランギス3の床、デルメゼ4のボム、シドーの行動周期に対応しています
時報を流せるので時間が見れない場合でも役に立ちます多分

サウンドはmp3で設定しているので差し替えれば変更することができます
start.mp3がスタート時のサウンド
record_a.mp3が通常(1回)記録時のサウンド（コールサファイア等）
record_b.mp3が周期記録時のサウンド（スクランブル等）
activation.mp3がギミック前の時報サウンドです

細かい使い方などはreadmeの使い方とconfigの設定で確認してください

- 導入方法

1. 右記のReleasesページから最新の XTimer_vX.X.X.zip をダウンロード
   https://github.com/syoya3214/XTimer/releases

2. zipを任意のフォルダに解凍する

3. 解凍したフォルダ内の XTimer.exe を実行する

※ Windows Defender などのウイルス対策ソフトが警告を出す場合がありますが、
   「詳細情報」→「実行」で起動できます（PyInstallerでビルドした自作EXEのため誤検知です）

- 使い方

StartKey（初期: F12キー）を押したタイミングでカウントダウンがスタートします（内部で動くので表示の時間はかわりません）
初期は09分59秒と19分59秒に設定してあるので戦闘開始直後にスタートしてください（ホットキーで変更可）

コールサファイアを確認したときにlog_key（初期: Insertキー）で指定したキーを入力すると起爆時間を記録します
一番近い起爆時間が画面に表示され、起爆直前になったら時報が再生されます

スクランブルサファイアやサークルカラミティを確認したら、log_key_loop（初期: Homeキー）を入力してください
入力するとその後の周期が全て記録されます
手動で記録する関係上、数秒ズレることがあるので後述のincrement_key、decrement_keyを使い調整してください

2回目の周期系（ブリリアントサファイア、グランドカラミティ等）を確認したら、log_key_loopB（初期: Endキー）を入力してください
ループAとループBは別々のキーで記録します

プリセットによってはloopA_stop_time / loopB_stop_timeが設定されており、
loopAまたはloopBを記録してから指定秒数後にループが自動停止します
（例：シドーはloopA記録から60秒後、loopB記録から90秒後に自動停止）

log_key_ex（初期: Deleteキー）で指定したキーを入力するとタイマーの右にconfigで設定した秒数が経ったときの時間が記録されます
バフ管理などに使えるかもしれません
activation_exが0の場合は押した瞬間の時刻を表示し、もう一度押すか次回記録まで表示が持続します

increment_key（初期: PageUPキー）、decrement_key（初期: PageDOWNキー）で時間の変更ができます
タイマーが停止しているか動作しているかで挙動が変わり、
タイマー停止中に入力するとタイマーの開始時間を10秒ずつ変更できます
これは開始時間の入力を忘れたりしてずらしたい時に使います
タイマーを開始後リセットを行うともとの開始時間に戻ります

タイマー動作中に入力すると周期の記録全体を1秒ずつずらすことができます
周期がずれたや入力が遅れた場合の調整に使えます

どちらも押しっぱなしで連続入力になります（負荷でたまにプログラム落ちるかも・・・）

ai_count_key_2（初期: F7キー）、ai_count_key_3（初期: F8キー）を使うと、
それぞれタイマーの上に12、123が切り替わるようになっています
何かのカウントに使えるかも知れません

sheet_switch_key（初期: F9キー）を使うとエクセルシート内のシートが切り替わるようになっています
HP変化などに合わせて使うといいかもしれません
たまにキーがきかなくなることがあるかもしれません
ちなみに表が要らなければプリセットで表を消すか、右クリックメニューから非表示にしてください

preset_switch_key（初期: ctrl+shift+tabキー）を押すとタイマーのプリセットが切り替わります
複数ボスで使い分けたい場合に

reset_position_key（初期: ctrl+shift+Rキー）を押すと配置のリセットが行えます
画面外のどこかに消えてしまった場合などに

コンフィグ
Config.iniを開いて編集してください
一部の項目はタイマーやテーブルの文字上を右クリックして編集することもできます

[Settings]

#Layout
opacity
画面透過の設定（不透明度）

timer_x
timer_y
table_x
table_y
起動時のタイマーとテーブル（xlsx）の位置（配置1）

timer_x_2
timer_y_2
table_x_2
table_y_2
タイマーとテーブル（xlsx）の位置（配置2）
それぞれ文字上を右クリックして配置1と配置2の切り替えと現在位置の保存が可能

table_align
テーブルの右揃え、左揃えを変更
table_visible
テーブルの表示（1）、非表示（0）の変更

ai_count_visible
AIカウントの表示（1）、非表示（0）の変更


#Font
timer_font_size
label_font_size
フォントサイズを変更可能

timer_font_color
timer_font_color_while_running
timer_font_color_ex
ai_count_font_color
フォントカラーを設定可能

outline_color
フォントの縁取りカラーを設定可能


#Sound
play_start_sound
スタート時のサウンド[start.mp3]の有無（true/false）

play_record_sound
LogKey入力時のサウンド[record_a.mp3][record_b.mp3]の有無（true/false）

play_activation_sound
起爆前の時報サウンド[activation.mp3]の有無（true/false）

sound_volume
サウンド全体のボリューム（0～100）

sound_delay
時報（activation.mp3）が始まるタイミングの設定（ミリセカンド）
mp3を差し替えたら変更いるかもしりません


#Hotkey
start_key
タイマーをスタートするときに押すキー
リセットするときはもう一度

log_key
サファイアボムなどの単発系ギミックを記録するときに押すキー

log_key_ex
タイマー右に一時的に記録をするときに押すキー
バフ管理やCT管理の補助に

log_key_loop
ループA（1回目の周期系）を記録するときに押すキー

log_key_loopB
ループB（2回目の周期系）を記録するときに押すキー
ループAと別キーで独立して記録できる

increment_key
decrement_key
時間を増減させるキー
タイマー停止中に入力するとタイマーの開始時間を変更できる
タイマー動作中に入力すると記録した時間をずらすことができる
記録全体が変更されるので適当に押してしまったときに調整したり、周期がズレたときに調整したり

ai_count_key_2
押すごとにタイマー上に1、2を順番に表示する
AIカウント補助に

ai_count_key_3
押すごとにタイマー上に1、2、3を順番に表示する
AIカウント補助に

sheet_switch_key
xlsxの表示しているシートを切り替え
HPで行動が切り替わったときなどに
たまにホットキーがきかなくなる不具合がある気がしなくもない

preset_switch_key
プリセットの切り替え
違うボスに行くときに

reset_position_key
うっかりタイマーなどを見失ったときに

[Preset]
[Preset1]、[Preset2]と複数増やしていけます
切り替えはHotkeyや右クリックメニューなどで

name
プリセット名

table_file
プリセットと合わせて読み込むxlsxファイルを指定
いらない場合は空欄かtable_visibleで非表示に

initial_time
カウントを開始するタイミングを設定できます
初期は09:59と19:59
開始は自動ではなく手動なので注意
押すのが遅れたらincrement_key、decrement_keyでスタート位置をずらして対応してください
分を1桁にすると表示くずれちゃうかもなんで09:59といった感じにしてください

activation_normal
記録からギミック発動までの時間(ms)
押す事に1回ずつ追加で記録

activation_ex
記録からCTやバフがオンオフする時間(ms)
押す事に前の記録は消えて更新される
0に設定すると押した時刻をそのまま表示し続ける（手動で消すまで残る）

activation_loopA
activation_loopB
記録からループギミックが起動するまでの時間(ms)
Aは1回目（サークルカラミティ等）、Bは2回目（グランドカラミティ等）と別々に設定できる
ルベランギスなら床出現まで5000～6000 msくらい、デルメゼなら爆弾起爆まで108500 msくらい

loopA_time
loopB_time
ギミックの発動周期
タイミングが何回目かによって変わる場合は,で追加して対応
ルベランギスの場合は交互

loopA_stop_time
loopB_stop_time
ループ記録から指定ミリ秒後にループを自動停止する時間(ms)
設定しない、または0の場合は自動停止しない
シドーなどループが一定回数しかない場合に便利
（例: loopA_stop_time = 60000 で記録から60秒後に停止）

log_label
タイマーの下に指定した文字や画像を表示できる
  \nか\\nが改行コードで各行ごとに文字か画像かどちらかを表示（文字+画像などは無理）
  .pngなど拡張子がついている場合は画像判定されるはず

log_label_ex
log_key_exで表示した追加タイマーの下に指定した文字や画像を表示できる
  \nか\\nが改行コードで各行ごとに文字か画像かどちらかを表示（文字+画像などは無理）
  .pngなど拡張子がついている場合は画像判定されるはず

loopA_labels
loopB_labels
ループ時のタイマー下に指定した文字や画像を表示できる
,で文字などを追加していくことでループの周期に合わせて切り替えられる
  \nか\\nが改行コードで各行ごとに文字か画像かどちらかを表示（文字+画像などは無理）
  .pngなど拡張子がついている場合は画像判定されるはず
